第1章 绪论
1.1 研究背景与意义
1.1.1 研究背景
森林火灾具有突发性强、蔓延速度快、处置窗口短的特点。监测链路一旦出现延迟，火情规模会在短时间内扩大，现场调度压力随之上升。林区点位分散、地形遮挡明显，单一手段往往难以覆盖全流程需求，预警系统需要同时满足“早发现、少误报、可追溯、可协同”四个条件。

当前林区值守仍大量依赖人工巡查和视频轮巡。人工方式在小范围内可行，面对多路视频并发时会出现注意力衰减，告警识别的一致性也会下降。监控设备已被广泛部署，视频数据持续产生，真正瓶颈转移到了“如何稳定识别”和“如何把识别结果转成可执行处置”。

视觉检测模型在火点、烟雾识别上具备实时性优势，YOLO 一类模型已被大量验证可用于流式目标检测。问题也很明确：夕阳反光、灯光干扰、雾霾和低清画面会触发误检，单模型结论直接驱动告警时，值守端会出现无效弹窗堆积，处置资源被占用。模型能力、现场噪声、业务流程之间的矛盾由此暴露出来。

该课题对应的系统场景由三类角色共同参与：操作员负责告警处置，主管负责设备和统计，管理员负责参数与权限。检测链路、告警推送、任务闭环、历史回溯需要被统一到同一平台内，否则信息会在系统边界处丢失，复盘依据无法稳定保留。

1.1.2 研究意义
工程意义体现在“检测—复核—处置—归档”闭环被落地。系统将前端视频巡检、后端推理、WebSocket 实时推送、SOP 处置流程连接为一条可执行链路，操作员接到告警后能够直接完成记录、上报和结果归档，流程被标准化，现场协同效率得到提升。

方法意义体现在双阶段判别策略。高置信度结果被快速上送，低置信度结果被静默归档，中间区间交由多模态模型复核并输出明确结论（真实火灾或误报）。计算资源会被优先分配到不确定样本，误报压力被压低，漏报风险通过保守兜底策略进行控制，实时性与准确性之间形成可调平衡。

管理意义体现在数据资产沉淀。告警记录、处置备注、设备状态、系统参数被结构化保存，统计看板可以直接输出趋势与复盘线索。值守经验不再只停留在个人层面，后续培训、考核、参数优化可以基于历史数据进行，系统运行质量能够被持续评估。

1.2 国内外研究现状
1.2.1 国内研究现状
国内对森林火灾监测的研究起步较早，研究路线大体经历了人工巡护、固定监控、图像识别、智能预警几个阶段。早期工作主要依赖瞭望塔值守、人工巡山和地面巡检，覆盖范围有限，信息反馈链路较长，火情发现时间容易受到地形、天气和人员经验的影响。监控设备逐步进入林区后，视频监控、红外成像、烟雾传感器等手段被用于辅助巡检，监测效率有所提高，但系统之间往往彼此分离，告警信息和后续处置记录缺少统一管理。

近几年，国内高校和科研机构将深度学习方法引入林火识别任务，研究重点集中在火焰、烟雾目标检测以及复杂场景下的误报抑制。YOLO、Faster R-CNN、SSD 等模型被用于林火图像识别，模型推理速度和检测精度较传统方法有明显改善。研究者普遍关注林区视频中光照剧烈变化、远距离小目标、烟雾形态不稳定等问题，并通过数据增强、改进网络结构、迁移学习等方式提升识别效果。此类研究对算法性能提升具有直接价值，但实际部署层面仍存在一些共性问题，例如模型输出结果偏向“检测是否存在疑似目标”，缺少面向业务处置的解释信息，误报后的人机协同流程也未被充分纳入系统设计。

工程应用方面，国内部分林业部门和企业已开始建设火情监测平台，功能通常覆盖视频接入、告警显示、设备管理和基础统计。平台化建设提升了监测的可视化水平，也使林区值守具备了一定的数字化特征。问题依旧明显：其一，多路视频同时在线时，人工仍需承担大量复核工作；其二，部分系统对误报场景处理不足，夕阳反光、灯光干扰、雾气漂移等容易被判为火情；其三，告警产生后的任务派发、处置闭环、历史回溯和权限管理常被拆散处理，业务链路不够完整。国内研究正在从“单点识别模型优化”转向“检测、复核、处置一体化平台建设”，这一变化为本系统的设计提供了现实依据。

1.2.2 国外研究现状
国外在森林火灾监测领域的研究更强调多源信息融合和实战部署能力。美国、加拿大、澳大利亚、西班牙等国家较早将卫星遥感、航空巡检、地面摄像头网络、气象数据和地理信息系统结合起来，用于构建区域级火情监测体系。卫星遥感适合大范围巡测，覆盖能力强，但时间分辨率和局部细节识别能力存在限制；地面视频监控和无人机巡检能够补足近距离连续观测这一环节。监测体系由单一设备转向多源协同已成为国外较常见的技术路径。

在算法层面，国外研究同样广泛采用卷积神经网络和目标检测模型处理火焰、烟雾识别问题，并开始重视时空信息建模。部分研究将视频连续帧、环境参数、风速风向以及地表特征共同纳入判别过程，试图减少静态图像识别带来的片面性。对误报来源的分析也更细，太阳反射、云层边缘高亮、工业烟气、车辆灯光等典型干扰因素通常会被单独建模或作为专门数据集的一部分处理。国外系统设计中还有一个明显特点，即强调决策支持而非只给出识别结果。火情等级、扩散趋势、资源调度建议、疏散路径分析等内容常被整合进监测平台，用于服务消防指挥和应急联动。

近年的国外研究开始引入更强的视觉语言模型和多模态推理框架，用于提升复杂场景解释能力。与传统检测模型相比，这类方法在语义理解、场景描述和复核说明上更具优势，能够给出“是否存在真实火情”“疑似误报来源是什么”“是否需要人工继续核查”等更贴近业务的问题回答。其局限也很清楚：计算成本较高，实时性控制更难，部署到边缘侧时对资源要求较严。国外研究已经从“识别有没有火”逐步转向“解释为什么判定为火、该采取什么行动”。这一趋势说明，火灾预警系统的竞争点已经不只停留在检测精度本身，业务闭环能力、人机协作效率和系统可落地性同样重要。

1.3 论文的组织结构
第一章为绪论，交代课题来源、研究背景、研究意义，并对国内外相关研究进行梳理，明确本课题所对应的问题场景与研究价值。

第二章介绍系统开发所使用的主要工具与关键技术。内容包括前端框架 Vue 3、后端框架 FastAPI、数据库访问技术 SQLAlchemy/SQLModel、YOLO 目标检测方法以及多模态大模型复核机制。该章为后续系统分析、设计与实现提供技术基础。

第三章为系统分析。围绕森林火灾预警平台的实际使用场景，从技术可行性、经济可行性和操作可行性三个方面展开说明，并对安全操作员、安全主管、系统管理员三类角色的功能需求进行拆解，同时给出系统在安全性、响应速度和多路并发处理方面的非功能需求。

第四章为系统设计。该章从整体架构入手，说明前后端分离模式下的系统组成，以及“YOLO 初检 + LLM 复核”的双阶段工作流。在此基础上，对认证与权限、视频巡检、语义复核、告警闭环等核心模块进行详细设计，并完成数据库概念结构设计与数据表设计。

第五章为系统实现。该章结合项目目录结构和核心代码实现过程，对管理员端、主管端、操作员端各模块进行说明，重点展示监控大屏、实时告警推送、任务处置、历史回溯、统计分析和设备管理等功能的落地方式。

第六章为系统测试。该章给出测试目的、测试环境与测试方法，并从功能测试和非功能测试两个层面验证系统运行情况，重点检查多路视频处理、权限隔离、告警流转、响应延迟和识别效果等内容。

第七章为系统展望与结论。该章对系统开发工作进行归纳，指出当前实现中仍然存在的限制，并给出后续可继续完善的方向。

1.4 本章小结
本章完成了课题研究背景、现实需求和研究意义的说明，并对国内外森林火灾监测与智能预警相关研究进行了梳理。现有研究已经证明，视觉检测技术能够提升火情识别效率，多源协同与智能复核正在成为火灾预警系统的重要发展方向。现阶段仍存在误报抑制不足、业务闭环不完整、人机协同效率偏低等问题。基于这些问题，后续章节将围绕系统分析、总体设计、模块实现和测试验证展开，对森林火灾预警平台的建设过程进行详细说明。
