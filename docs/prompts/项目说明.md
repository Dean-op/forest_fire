# 森林火灾预警系统项目说明

## 1. 项目概况

本项目是一套面向林区值守场景的森林火灾预警系统，论文题目对应为“基于深度学习与大语言模型的森林火灾预警系统设计与实现”。系统采用前后端分离架构：前端提供监控大屏、任务处置、统计分析和系统管理页面；后端负责用户认证、视频流处理、YOLO 目标检测、告警记录、LLM 复核、WebSocket 推送和业务数据管理。

项目的核心目标不是替代人工值守，而是通过视频巡检、模型初筛和告警闭环降低人工轮巡压力，使值守人员能够更快发现疑似火情、查看证据截图、完成处置记录，并在后续通过历史告警和统计数据进行复盘。

## 2. 技术栈

### 2.1 后端技术

- Web 框架：FastAPI
- 数据库访问：SQLModel、SQLAlchemy
- 数据库：由 `DATABASE_URL` 环境变量配置，可连接 MySQL 等关系型数据库
- 视频处理：OpenCV
- 目标检测：Ultralytics YOLO
- 大模型复核：通过 `httpx` 调用多模态大模型接口，配置中预留 SiliconFlow / Qwen-VL 相关参数
- 实时通信：WebSocket
- 权限认证：JWT、python-jose、passlib、bcrypt
- 静态资源：FastAPI StaticFiles 挂载 `uploads/`，用于访问告警截图

### 2.2 前端技术

- 前端框架：Vue 3
- 构建工具：Vite
- UI 组件库：Element Plus
- 状态管理：Pinia
- 路由管理：Vue Router
- 数据可视化：ECharts
- 网络请求：Axios

## 3. 系统总体架构

系统采用“前端页面层 - API 通信层 - 后端业务层 - 数据与文件存储层”的结构。

前端页面层负责展示各角色的业务界面，包括实时监控大屏、任务面板、历史告警、统计看板、视频源管理、交接班日志、用户管理、系统参数、公告和备份页面。前端通过 Axios 调用 REST API，通过 WebSocket 接收实时告警，通过 MJPEG 地址展示后端输出的视频流。

后端业务层由多个路由模块组成。`auth` 模块负责登录、注册、当前用户信息和密码修改；`stream` 模块负责视频源读取、YOLO 推理、截图保存、分级判断和 LLM 复核；`alerts` 模块负责告警查询、处理、联动、归档、导出和批量操作；`supervisor` 模块负责统计、摄像头、设备日志、抓拍配置和交接班日志；`admin` 模块负责用户、系统参数、公告、系统日志和备份相关接口；`ws` 模块负责维护 WebSocket 连接并广播告警消息。

数据层以 SQLModel 模型为基础，主要包括用户、告警、摄像头、摄像头日志、抓拍配置、交接班日志、系统配置、公告和系统日志等实体。文件层主要保存演示视频、告警音频和火情截图等资源。

## 4. 核心业务流程

### 4.1 用户认证与角色控制

系统设置操作员、主管和管理员三类角色。用户登录后，后端签发 JWT，前端保存 token 并在请求拦截器中自动携带。路由层根据用户角色控制菜单和页面访问范围。

操作员主要负责查看告警、处理任务和回溯历史事件；主管主要负责查看统计数据、管理视频源、查看设备状态和填写交接班日志；管理员主要负责用户管理、系统参数配置、公告管理、系统日志查看和数据备份。

### 4.2 视频流接入与 YOLO 巡检

后端通过 OpenCV 读取摄像头或本地演示视频，并以 MJPEG 形式输出给前端监控大屏。系统不会对每一帧都执行推理，而是根据系统参数中的 `yolo_interval` 按间隔触发检测，以降低运行压力。

YOLO 检测任务通过异步任务执行，检测结果会在视频画面中绘制识别框，并返回最高置信度。系统根据双阈值策略对检测结果进行分流：高于高阈值的结果直接生成待核实告警；低于低阈值的结果记录为低风险并静默归档；处于中间区间的结果进入 LLM 二次复核流程。

### 4.3 LLM 复核与告警推送

当检测置信度处于中间区间时，系统会保存当前帧截图，并调用多模态大模型接口进行语义复核。复核结果被整理为统一文本格式，包含“复核结论、风险级别、处置建议”三部分。

如果复核结果倾向于真实火情或需要人工确认，系统将告警推送给前端；如果复核结果为低风险或误报，则可进入归档流程。实时告警通过 WebSocket 推送，前端监控大屏在收到消息后展示告警中心、截图、置信度和处置建议。

### 4.4 告警闭环处置

告警记录进入系统后，操作员可以在任务面板中查看待处理告警，结合截图、YOLO 置信度和 LLM 分析文本进行判断。处理时可标记真实火灾、误报、联动处置或已处理，并填写备注。历史事件页面支持查看不同状态的告警记录，并提供 CSV 导出能力。

该流程形成“检测 - 复核 - 推送 - 处置 - 归档 - 回溯”的闭环，适合在论文中作为系统业务流程和详细实现章节的主线。

## 5. 功能模块说明

### 5.1 公共基础模块

- 登录与注册：支持用户名密码登录，新用户默认注册为操作员角色。
- JWT 鉴权：后端校验 token，前端在请求中自动携带认证信息。
- 个人中心：展示当前账号信息并支持修改密码。
- 角色路由：前端基于角色控制页面可见性，后端关键接口也设置角色校验。

### 5.2 安全操作员模块

- 实时监控大屏：展示多路视频画面，接入后端输出的检测视频流。
- 告警中心：通过 WebSocket 接收实时告警并展示告警详情。
- 我的任务：查看待核实、待处置告警，提交处理结果和备注。
- 历史回溯：查看历史告警记录、截图和复核文本。
- 数据导出：导出告警记录 CSV，便于后续分析和归档。
- 公告查阅：查看管理员发布的通知和 SOP 内容。

### 5.3 安全主管模块

- 统计分析：展示告警总量、设备在线率、趋势数据和告警状态分布。
- 视频源管理：维护摄像头名称、地址、位置、状态和 AI 检测开关。
- 设备日志：查看设备上线、掉线、恢复等记录。
- 抓拍配置：维护摄像头抓拍相关参数。
- 交接班日志：记录和查看值守交接信息。

### 5.4 系统管理员模块

- 用户管理：创建用户、修改角色、启用或禁用账号。
- 系统参数：配置 YOLO 检测间隔、推理缩放、高低阈值、告警冷却时间、LLM 接口地址、API Key 和模型名称等。
- 公告与 SOP 管理：发布通知和作业流程说明。
- 系统日志：查看登录、配置变更、告警处理等操作记录。
- 数据备份：提供演示级备份操作与备份文件下载接口。

## 6. 数据模型概览

项目中的主要模型包括：

- `User`：用户表，保存用户名、密码哈希、角色、启用状态和创建时间。
- `Alert`：告警表，保存告警时间、截图路径、YOLO 置信度、摄像头名称、LLM 复核结果、处理状态和备注。
- `Camera`：摄像头表，保存设备名称、RTSP 地址、位置、在线状态和 AI 检测开关。
- `CameraLog`：设备日志表，保存摄像头事件和发生时间。
- `CaptureConfig`：抓拍配置表，保存抓拍开关、间隔和原图保存配置。
- `ShiftLog`：交接班日志表，保存值班人员、班次时间和交接内容。
- `SystemConfig`：系统参数表，采用键值形式保存 AI、LLM 和通用配置。
- `Announcement`：公告表，保存通知或 SOP 内容。
- `SystemLog`：系统日志表，保存用户操作、详情、IP 和时间。

这些模型可以支撑论文中的数据库设计、功能模块实现和系统测试章节。

## 9. 写论文时建议避免的表述

为了降低论文内容与项目代码不一致的风险，建议避免以下写法：

- 不写“已完成生产环境部署”“大规模并发运行”“长期在线运行验证”等代码仓库无法证明的结论。
- 不写具体性能提升百分比、误报率下降比例、识别准确率等未经过正式实验统计的数据。
- 不把本系统描述成完全自动化的无人值守系统，应表述为“辅助值守人员发现、复核和处置告警”。
- 不夸大 LLM 的作用。更合适的说法是“对中等置信度样本进行语义复核并生成处置建议”，而不是“完全替代人工判断”。
- 不把备份、RTSP 接入、设备日志等演示性功能写成复杂生产级运维平台能力。
