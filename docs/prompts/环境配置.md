# 森林火灾预警系统 — 环境配置指南

---

## 一、后端环境（Python + FastAPI）

### 1.1 项目目录
```
forest_fire_backend/
├── app/
│   ├── __init__.py
│   ├── main.py            # FastAPI 入口
│   ├── config.py           # 数据库 & 全局配置
│   ├── database.py         # SQLModel 引擎与会话
│   ├── models/             # 数据模型
│   │   ├── __init__.py
│   │   └── user.py
│   ├── routers/            # 路由模块
│   │   ├── __init__.py
│   │   └── auth.py
│   ├── utils/              # 工具函数（JWT等）
│   │   ├── __init__.py
│   │   └── security.py
│   └── seeds.py            # 种子数据脚本
├── uploads/                # 静态资源（截图等）
├── pyproject.toml          # uv 项目配置与依赖
├── uv.lock                 # uv 锁定文件
└── .env                    # 环境变量（不入库）
```

### 1.2 核心依赖（通过 uv 管理）
```bash
uv add fastapi uvicorn[standard] sqlmodel pymysql cryptography python-jose[cryptography] passlib[bcrypt] python-dotenv python-multipart
```

### 1.3 环境变量（.env）
```env
DATABASE_URL=mysql+pymysql://root:your_password@localhost:3306/forest_fire
SECRET_KEY=your-super-secret-key-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
```

### 1.4 启动命令
```bash
cd forest_fire_backend
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 二、前端环境（Vue 3 + Vite）

### 2.1 创建命令
```bash
npm create vite@latest forest_fire_frontend -- --template vue
cd forest_fire_frontend
npm install
```

### 2.2 核心依赖
```bash
npm install vue-router@4 pinia axios element-plus @element-plus/icons-vue echarts
```

### 2.3 项目目录（初始化后）
```
forest_fire_frontend/
├── public/
├── src/
│   ├── api/                # Axios 封装与接口定义
│   ├── assets/             # 静态资源
│   ├── components/         # 公共组件
│   ├── layouts/            # 布局组件
│   ├── router/             # Vue Router 配置
│   ├── stores/             # Pinia 状态管理
│   ├── views/              # 页面视图
│   ├── App.vue
│   └── main.js
├── index.html
├── vite.config.js
└── package.json
```

### 2.4 启动命令
```bash
cd forest_fire_frontend
npm run dev
```

## 三、数据库

### 3.1 MySQL 8.0
```sql
CREATE DATABASE IF NOT EXISTS forest_fire
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

> 无需手动建表，后端启动时 SQLModel 自动根据模型 `create_all()` 生成。

## 五、前端代理配置（vite.config.js）

开发环境下将 `/api` 请求代理到后端，避免跨域问题：

```js
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
      '/ws': {
        target: 'ws://localhost:8000',
        ws: true,
      },
      '/static': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
})
```
