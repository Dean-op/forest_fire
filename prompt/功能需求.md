



**全局基础模块（门面与基石）**

* 
**登录与注册**：提供安全登录入口，基于 JWT 实现无状态校验 。


* 
**权限路由控制**：基于角色的访问控制，不同角色（操作员/主管/管理员）登录后，左侧显示的菜单项不同 。（实现策略：前端 Vue Router 配合 `v-if` 极简控制）。


* 
**个人中心**：管理个人账户，如修改密码 。



**核心看点：双核智能预警大屏（答辩时的主战场）**

* 
**多路视频网格视图**：同时浏览多路监控 。（实现策略：前端放 4 个格子，3 个用 HTML5 播放预置 MP4 假装是正常监控，1 个连接后端真实展示 YOLO 实时画框检测的视频流）。


* 
**YOLO 实时视觉巡检**：毫秒级响应，发现火情或烟雾后立即在视频流上绘制警示框 。


* 
**LLM 语义复核机制**：YOLO 判定疑似火灾后，自动截取高清图片并异步发送至大模型（如 Qwen-VL），获取“是否为真实火灾”的专家级文本判定 。


* 
**实时告警弹窗**：通过 WebSocket 技术，将双核确认的火情第一时间推送到大屏，伴随明显的视觉和声音提示 。



**安全操作员功能（一线值守视角）**

* 
**我的任务面板**：只显示状态为“待处理”的告警，聚焦核心工作 。（实现策略：一个简单的过滤查询表格）。


* 
**历史事件回溯**：查看所有历史告警，查看告警时留存的证据 。（实现策略：把开题报告里的“自动录制视频”降维为“高清红框截图 + LLM 详细分析报告”，避免服务器被视频存取搞崩，答辩效果一样震撼）。


* 
**告警事件处理**：对告警进行响应（确认为真火灾或误报），填写简易的处理报告/备注 。


* 
**SOP与公告查阅**：查看管理员发布的标准作业程序和系统级通知 。



**安全主管功能（数据与设备管理视角）**

* **统计分析仪表盘**：利用 ECharts 查看全局宏观数据统计 。（实现策略：画几个好看的饼图和折线图，展示今日告警数、真假警报比例、设备在线率等）。


* 
**视频源管理**：增、删、改、查所有 RTSP 监控设备 。（实现策略：一个 CRUD 表格，输入框允许填 RTSP，但后端统配映射到本地演示 MP4 上）。


* 
**摄像头状态与日志监控**：查看设备“在线/离线”状态，以及掉线/恢复历史记录 。（实现策略：数据库给设备表加个 status 字段，默认全写在线，日志随便生成几条即可）。


* **事件抓拍配置**：配置告警触发时的抓拍逻辑 。（实现策略：替代报告里的“录制前后N秒”，改为前端的一个配置选项表单，哪怕后端不实际读取，只要能存能改就行）。


* 
**告警事件导出**：一键将数据导出为 Excel/CSV 格式 。（实现策略：前端引入类似 `xlsx` 库或后端用 `pandas` 直接返回文件流，代码极短）。


* 
**交接班日志**：查看或填写交接班工作记录 。（实现策略：一张极简的 CRUD 文本表）。



**系统管理员功能（底层配置视角）**

* 
**用户权限管理**：创建、禁用账号，分配“主管”和“操作员”角色 。（实现策略：经典的 User 表增删改查）。


* **系统参数配置**：配置 AI 算法参数（如 YOLO 置信度阈值）、大语言模型的 API Key 等 。（实现策略：做一个高大上的参数设置页，将参数存入数据库或 Redis，推理时读取）。


* **SOP与公告管理**：增、删、改、查标准作业程序文章和通知 。（实现策略：带个简单富文本编辑器的 CRUD）。


* 
**系统日志查看**：审计和排错日志 。（实现策略：捕获 FastAPI 的运行日志展示到前端，或者只做一个简单的接口调用记录表）。


* **数据备份**：管理数据库备份 。（实现策略：前端放个“一键备份”按钮，后端执行一个 shell 脚本备份 MySQL，或者干脆只做个前端进度条和“备份成功”的提示应付演示）。
