# 森林火灾预警系统 (Forest Fire Early Warning System)

## 1. 项目概况
本项目是一个基于机器视觉大模型与边云协同架构的**森林火灾智能预警平台**。系统通过挂载前端摄像头的视频流数据，在边缘侧/本地进行高频的轻量级目标检测（YOLO），并在疑似告警时自动将关键帧上报云端多模态大语言模型（Qwen-VL）进行二次复核，最终将告警结果通过 WebSocket 实时推送至管理大屏，形成从“发现”、“研判”、“告警”到“处置”的完整业务闭环。

## 2. 技术栈架构
### 后端技术栈 (Backend)
- **Web 框架**: FastAPI (高性能、全异步、自带 Swagger 文档)
- **ORM 与数据库**: SQLModel + SQLAlchemy + MySQL / SQLite
- **环境隔离**: `uv` (新一代极致极速的 Python 包管理器)
- **AI 视觉引擎**: Ultralytics YOLO (实时视频帧对象检测)
- **云端大模型集成**: 硅基流动 SiliconFlow (接入 `Pro/Qwen/Qwen2.5-VL-7B-Instruct` 视觉大模型)
- **依赖库**: `opencv-python` (视频流编解码), `asyncio` & `websockets` (实时高并发双向通信), `httpx` (异步大模型 API 调用), `PyJWT` (安全认证)

### 前端技术栈 (Frontend)
- **核心框架**: Vue 3 (Composition API, `<script setup>`) + Vite (极速构建)
- **UI 组件库**: Element Plus (现代化且功能丰满的企业级 PC 端组件)
- **状态管理**: Pinia (轻量、TS 友好)
- **路由控制**: Vue Router 4 (支持动态路由与路由守卫)
- **数据可视化**: Apache ECharts (丰富炫酷的数据报表仪表盘)
- **网络请求**: Axios + 本地 Token 拦截器认证

## 3. 核心功能与亮点

### 🌲 核心亮点：边云协同级联推理系统 (Cascaded Inference System)
系统摒弃了低效的“每帧送云端大模型”或“纯靠小模型易误报”的单点方案，开创性地实现了**置信度三级分流与时间间隔抽帧机制**：
1. **防冗余抽帧**: 在视频流平滑推送到前端（~30fps）的同时，后台守护任务仅根据 `yolo_interval` (默认3秒) 从视频流中抽帧检测，极大地节省了本地 GPU/CPU 算力。
2. **三级置信度分流**:
   - **高确信度 (≥80%)**: YOLO 本地直接判定火情，0秒延迟生成最高级告警。
   - **疑似火情 (50%~79%)**: 立即截取当前帧画面，走独立的异步事件循环调用 Qwen-VL 进行二次多维复核，消除夕阳红、红旗等误报情况，视频流全程不卡顿。
   - **低置信度 (<50%)**: 视为环境噪点，直接忽略。
3. **全局告警冷却 (Cooldown)**: 任意节点触发告警后，立即进入 `alert_cooldown` (10秒) 的静默期，从根源上防止 API 资费爆炸和前端弹窗风暴。

### 💻 业务模块 (按角色)
#### 操作员工作台 (Operator Console)
- **监控大屏**: 实时渲染四宫格/多宫格联动监控流。AI 识别引擎自动在画面上叠加红/黄双色检测框。
- **即时告警弹窗**: 触发火灾时，基于 WebSocket 瞬间收到告警，展示起火时间点的高清分析截图及模型分析结果。
- **任务面板与历史纪要**: 将产生的告警进行快速的标记归档（真实火灾/误报），支持条件筛选与一键数据包 CSV 导出/批量删除。

#### 主管仪表盘 (Supervisor Dashboard)
- **数据总览 Stat**: 精美直观的 ECharts 统计大盘（总设备数工作情况、日均告警走势图、真伪占比饼图等）。
- **设备管理**: 对全场的监控摄像头进行增删改查。**支持针对于单路摄像头的 "AI 智能检测" 独立功能启停逻辑**。
- **交接班日志**: 在班操作记录的历史存档留痕。

#### 行政/超级管理员 (System Administrator)
- **系统参数配置**: 将 YOLO抽帧率、置信度双阈值、冷却时间、以及硅基流动的 API 秘钥等敏感关键参数，全部通过直观的 Tabs 界面开放给前台管理。随时热更新，无需重启系统。
- **权限与用户管理**: 基于 JWT 的多角色用户创建。
- **数据备份**: （预留）系统级的数据快照下载保护。

## 4. 目录结构精要

```text
forest_fire/
├── forest_fire_backend/          # 后端项目 (FastAPI)
│   ├── app/
│   │   ├── main.py               # 核心入口 & 中间件 & 路由挂载
│   │   ├── database.py           # SQLModel 引擎注册
│   │   ├── config.py             # 读取 .env
│   │   ├── seeds.py              # 初始化启动的 Mock 假数据与全局系统参数
│   │   ├── models/               # 数据模型映射目录 (auth/alerts/admin/supervisor...)
│   │   └── routers/              # 逻辑控制分层 (核心推流在 stream.py)
│   └── uploads/                  # 历史发生火灾瞬间告警截图归档保存点
│
├── forest_fire_frontend/         # 前端项目 (Vue 3)
│   ├── src/
│   │   ├── api/                  # Axios 封装的统一出口 index.js
│   │   ├── components/           # 复用级别的小组件
│   │   ├── layouts/              # 主框架包络组件 MainLayout.vue (含侧边栏与头部菜单)
│   │   ├── router/               # 路由表定义与导航守卫机制
│   │   ├── store/                # Pinia 状态层，管理 user 的信息持久化
│   │   └── views/                # 各大功能核心页面 (Dashboard, Stats, History...)
│   └── vite.config.js            # 工程编译与反向代理 Proxy 层配置
│
└── prompt/                       # 工程全局 Prompt 指令与开发架构计划集
```

## 5. 项目运行快速指南

### 启动后端
```bash
cd forest_fire_backend
# 安装 uv (如果尚未安装) : pip install uv
uv sync
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 启动前端
```bash
cd forest_fire_frontend
npm install
npm run dev
```

> **注意：** AI 系统中的 YOLOv8/11 系列权重为 `app/models/best.pt`，测试运行的基准数据流源文件在后端根目录被默认指为 `demo.mp4`，这两份数据源已被隔离在 Git 推送范畴之外以控制主仓库体积。
