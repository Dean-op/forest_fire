# 森林火灾预警系统 — 分阶段开发计划

> **目标**：以最精简的代码量，交付一套可演示、可答辩的 MVP 系统。  
> **原则**：每个阶段产出可独立运行的增量，前一阶段不完成不进入下一阶段。

---

## 第一阶段：项目骨架与用户认证

**目标**：搭建前后端项目结构，打通登录流程，实现角色路由隔离。

### 后端
- 初始化 FastAPI 项目结构（`main.py`, 路由目录, 模型目录, 配置文件）
- SQLModel 定义 `User` 模型（含 `role` 字段：`operator` / `supervisor` / `admin`）
- 配置 CORS Middleware，允许所有来源跨域请求
- MySQL 数据库连接与建表
- 预置三个默认账号（各角色一个）
- 建库建表后随机生成模拟种子数据（告警记录、设备信息、日志等），用于后续各阶段功能测试
- JWT 登录接口（`POST /api/auth/login`）
- 获取当前用户信息接口（`GET /api/auth/me`）
- 修改密码接口（`PUT /api/auth/password`）

### 前端
- Vite + Vue 3 项目初始化，集成 Element Plus、Vue Router、Pinia、Axios
- 登录页面（表单 + JWT 存储）
- 全局路由守卫（未登录跳转登录页）
- 基于角色的侧边栏菜单（`v-if` 控制菜单项可见性）
- 主布局骨架（顶栏 + 侧边栏 + 内容区）
- 个人中心页面（展示用户信息 + 修改密码）

### 交付验收标准
- [x] 三个角色账号均可登录
- [x] 不同角色看到不同的菜单
- [x] 修改密码可用
- [x] 未登录访问自动跳转

---

## 第二阶段：智能监控大屏（核心亮点）

**目标**：实现 YOLO 实时检测 + Qwen-VL 语义复核 + WebSocket 告警推送的完整链路。

### 后端
- YOLO 推理模块：读取本地 MP4 → 逐帧检测 → 标注画框 → MJPEG 视频流输出
- 火情截图保存逻辑（检测到疑似火灾时保存当前帧到本地）
- Qwen-VL 异步复核模块：将截图发送至阿里云 Qwen-VL API，获取文本判定
- WebSocket 端点（`/ws/alerts`）：向前端实时推送告警事件
- 告警事件入库（`Alert` 模型：时间、截图路径、YOLO 置信度、LLM 判定结果、状态）
- 配置 FastAPI 的 StaticFiles 挂载，将 `uploads/` 目录挂载到 `/static` 路由，使前端可通过 `http://localhost:8000/static/xxx.jpg` 访问截图等静态资源

### 前端
- **监控大屏页面**（所有角色可见）
  - 四宫格视频墙布局
    - 左上：接入后端 MJPEG 实时 YOLO 检测流
    - 其余三路：`<video>` 标签播放预置 MP4
  - WebSocket 连接管理（自动重连）
  - 实时告警弹窗（声音 + 视觉动效）
  - 告警信息卡片（截图缩略图 + YOLO 置信度 + LLM 分析摘要）

### 交付验收标准
- [x] 打开大屏能看到 4 路视频画面
- [x] YOLO 实时画框可见
- [x] 检测到火情后弹窗告警
- [x] 弹窗中包含 LLM 二次判定结果

## 第三阶段：告警闭环与操作员工作台

**目标**：让告警事件可被处理、可被回溯，形成完整的业务闭环。

### 后端
- 告警列表查询接口（支持状态筛选：待处理 / 已确认 / 误报）
- 告警处理接口（操作员提交处理结果和备注）
- 告警详情接口（返回截图 + LLM 分析报告全文）
- 告警数据导出接口（CSV/Excel 文件流）

### 前端
- **我的任务面板**（操作员角色）
  - 仅展示"待处理"告警的表格
  - 点击可查看详情（截图 + LLM 报告）
  - 处理操作：标记为"真实火灾"或"误报"，填写备注
- **历史事件回溯**（操作员角色）
  - 全量告警记录表格（含时间范围筛选、状态筛选）
  - 详情弹窗：高清标注截图 + LLM 详细分析报告

### 交付验收标准
- [x] 操作员可看到并处理待办告警
- [x] 处理后状态实时更新
- [x] 历史记录可查、可筛选
- [x] 数据可导出为 Excel

## 第四阶段：主管仪表盘与设备管理

**目标**：为主管角色提供数据统计大盘和设备管理能力。

### 后端
- 统计数据聚合接口（今日告警数、真/假警报比例、设备在线率、近 7 天趋势）
- 视频源 CRUD 接口（`Camera` 模型：名称、RTSP 地址、状态、位置描述）
- 设备日志接口（模拟的上下线记录）
- 抓拍配置 CRUD 接口（`CaptureConfig` 模型）
- 交接班日志 CRUD 接口（`ShiftLog` 模型）

### 前端
- **统计分析仪表盘**（主管角色）
  - ECharts 饼图：真实火灾 vs 误报占比
  - ECharts 折线图：近 7 天告警趋势
  - 数字卡片：今日告警数、设备在线率等 KPI
- **视频源管理**（主管角色）
  - CRUD 表格 + 新增/编辑对话框
- **设备状态监控**（主管角色）
  - 设备列表（在线/离线状态标签）+ 日志时间线
- **抓拍配置**（主管角色）
  - 配置表单（参数存取即可）
- **交接班日志**（主管角色）
  - CRUD 表格 + 文本编辑

### 交付验收标准
- [x] 仪表盘图表正确渲染
- [x] 设备增删改查可用
- [x] 交接班日志可写可查

## 第五阶段：系统管理与收尾打磨

**目标**：完成管理员专属功能，全局 UI 打磨，确保演示流畅。

### 后端
- 用户管理 CRUD 接口（创建/禁用账号、分配角色）
- 系统参数配置接口（YOLO 置信度阈值、API Key 等键值对读写）
- SOP 与公告 CRUD 接口（`Announcement` 模型：标题、富文本内容、发布时间）
- 系统日志查询接口（简易的接口调用记录）
- 模拟数据库备份接口（返回成功提示即可）

### 前端
- **用户权限管理**（管理员角色）
  - 用户列表表格 + 新增/编辑/禁用操作
- **系统参数配置**（管理员角色）
  - 高端感的参数设置页面（分组表单）
- **SOP 与公告管理**（管理员角色）
  - CRUD 列表 + 简单富文本编辑器
- **SOP 与公告查阅**（操作员角色）
  - 只读列表 + 详情查看
- **系统日志**（管理员角色）
  - 日志列表表格（时间 + 操作 + 用户）
- **数据备份**（管理员角色）
  - 一键备份按钮 + 进度条动画 + 成功提示
- **全局打磨**
  - 统一 Loading / Error 状态处理
  - 响应式适配检查
  - 演示用种子数据填充

### 交付验收标准
- [x] 管理员可管理用户和角色
- [x] 系统参数可配置并持久化
- [x] SOP 发布后操作员可查阅
- [x] 备份按钮点击有完整交互反馈
- [x] 全系统无明显 UI 瑕疵，演示流畅

## 阶段依赖关系

```
第一阶段（骨架与认证）
    └──▶ 第二阶段（智能监控大屏）
            └──▶ 第三阶段（告警闭环）
                    ├──▶ 第四阶段（主管仪表盘）
                    └──▶ 第五阶段（系统管理）
```

> **说明**：第四、五阶段互相独立，可视时间灵活调整顺序。第三阶段依赖第二阶段的告警数据，因此必须在大屏之后。
